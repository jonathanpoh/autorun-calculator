A web-based calculator for planning astronomy imaging times.
Completely client-side, no server backend needed.
Simple clean UI with minimal javascript frameworks, no build process. Static HTML, hand-rolled semantic CSS (no framework), vanilla JS.

Calculator functions:
Calculate the number of frames needed, given the imaging duration (either in decimal hours or minutes. Allow input of an 'overhead' percentage, due to dithering, autofocus, meridian flips, default 15%. Another field is the exposure time, dropdown with the most common ones, 60s 120s 180s 300s.
